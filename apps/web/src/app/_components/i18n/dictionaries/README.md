# Структура модульных словарей

Этот каталог содержит словари, разбитые по доменам для лучшей организации и поддержки.

## Структура

Каждый домен имеет три файла для трех языков:
- `{domain}.ky.ts` - киргизский язык
- `{domain}.ru.ts` - русский язык  
- `{domain}.en.ts` - английский язык

## Домены

### Основные домены:
- `auth.*` - авторизация и вход
- `booking.*` - публичный поток бронирования
- `home.*` - главная страница
- `header.*` - шапка сайта
- `footer.*` - подвал сайта

### Дашборд и управление:
- `dashboard.*` - дашборд владельца бизнеса
- `staff.*` - управление сотрудниками
- `services.*` - управление услугами
- `branches.*` - управление филиалами
- `finance.*` - финансовая статистика

### Административная панель:
- `admin.*` - административная панель (можно разбить на подмодули: admin/users, admin/categories, admin/businesses и т.д.)

### Общие:
- `common.*` - общие ключи (loading, noData, ok, error и т.д.)
- `datePicker.*` - компоненты выбора даты
- `bookings.*` - управление бронированиями
- `cabinet.*` - личный кабинет клиента
- `review.*` - отзывы
- `map.*` - карты
- `profile.*` - профиль пользователя
- `staffCabinet.*` - кабинет сотрудника
- `staffFinance.*` - финансы сотрудника

## Паттерн создания модуля

```typescript
// auth.ky.ts
export const authKy = {
    'auth.title': 'Kezek',
    'auth.subtitle': '...',
    // ... остальные ключи
} as const;
```

## Сборка словарей

Все модули собираются в `index.ts` с помощью функции `mergeDictionaries()`:

```typescript
export const ky: Translations = mergeDictionaries(
    authKy,
    bookingKy,
    dashboardKy,
    // ... остальные модули
);
```

## Миграция

1. Создать модули для домена (ky, ru, en)
2. Добавить импорты в `dictionaries/index.ts`
3. Добавить модули в `mergeDictionaries()` для каждого языка
4. После миграции всех доменов обновить `LanguageProvider.tsx` для использования модулей
